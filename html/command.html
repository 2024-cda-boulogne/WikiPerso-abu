Cette page vous permettra d'en savoir plus sur les commandes Git les plus utilisées

Les commandes Git

#1 git init

La commande git init nous permet de créer un nouveau référentiel Git. C'est la première commande pour démarrer un nouveau projet dans un référentiel GitHub. Allez dans le répertoire contenant vos fichiers de projet et exécutez la commande git init. Un sous-répertoire caché .git sera ajouté à celui-ci.

$ git init

Vous pouvez également fournir un nom de référentiel avec la commande git init.

$ git init 'nom'

#2 git clone

git clone crée une copie de travail locale du code source à partir d'un référentiel distant. Lorsque vous clonez un référentiel, le code sera automatiquement téléchargé sur votre machine. Cette commande ajoutera l'emplacement d'origine en tant qu'emplacement distant afin que vous puissiez récupérer les modifications et y pousser des changements si vous avez l'autorisation.

$ git clone 'url'

#3 git branch

La commande git branch nous permet d'ajouter une nouvelle branche à une branche existante, de visualiser toutes les branches existantes et de supprimer une branche.

Créer une nouvelle branche localement :

$ git branch 'nom branch'

Lister toutes les branches locales et distantes :

$ git branch -a

Afficher toutes les branches et voir sur quelle branche vous travaillez actuellement :

$ git branch ou $ git branch --list

Supprimer une branche :

$ git branch -d 'nom branch'

Renommer une branche :

git branch -m 'nom branch' 'nouveau nom branch'

#4 git checkout

La commande git checkout nous permet de passer à une branche existante ou de créer et de passer à une nouvelle branche. Pour ce faire, la branche vers laquelle vous souhaitez passer doit être présente sur votre système local et les modifications dans votre branche actuelle doivent être validées ou mises de côté avant de procéder au changement. Vous pouvez également utiliser cette commande pour vérifier les fichiers.

Passer à une branche existante :

git checkout 'nom branch'

Créer et passer à une nouvelle branche :

git checkout -b 'nom branch'

#5 git add

La commande git add ajoute vos modifications dans un fichier à la zone de préparation où vous pouvez comparer votre version locale et la version dans le référentiel distant. Avant de valider votre fichier nouvellement créé ou modifié, il doit être ajouté à la zone de préparation en utilisant la commande git add.

Ajouter des fichiers spécifiques :

$ git add 'nom fichier 1' 'nom fichier 2'

Ajouter tous les fichiers nouveaux, modifiés et supprimés :

$ git add -A

Ajouter les fichiers modifiés et supprimés :

$ git add -u

#6 git commit

git commit enregistre les modifications dans votre référentiel local. Chaque fois que vous validez vos modifications de code, vous devez inclure une brève description des changements effectués. Ce message de validation aide les autres à comprendre les modifications qui ont été apportées.

Valider tous les fichiers ajoutés avec la commande git add et les fichiers qui ont été modifiés depuis lors :

$ git commit -a

Valider les fichiers avec un message :

$ git commit -m “message commit”

Vous pouvez remplacer les deux commandes précédentes par la commande suivante :

$ git commit -am “message commit”

Modifier la dernière validation avec les dernières modifications comme une nouvelle validation :

$ git commit --amend -m “message commit"

#7 git push

git commit enregistre les modifications dans votre référentiel local. Chaque fois que vous validez vos modifications de code, vous devez inclure une brève description des changements effectués. Ce message de validation aide les autres à comprendre les modifications qui ont été apportées.

$ git push or $ git push 'remote' 'nom branch'

La commande git push envoie les modifications de fichiers validées du référentiel local vers le référentiel distant afin que d'autres puissent les utiliser. Elle créera également une branche nommée dans le référentiel distant si elle n'existe pas.

$ git push --set-upstream 'remote' 'nom banch' ou git push -u origin 'nom branch'

#8 git pull

git pull récupère les dernières modifications téléchargées depuis le serveur distant dans le référentiel local afin que vous puissiez disposer des dernières mises à jour de vos coéquipiers.

$ git pull ou $ git pull 'remote' ou $ git pull 'remote' 'nom branch'

#9 git merge

La commande git merge fusionne votre branche avec la branche parente. La branche parente peut être soit une branche de développement, soit une branche principale (master) selon votre flux de travail. Elle créera automatiquement une nouvelle validation s'il n'y a pas de conflits. Avant d'exécuter la commande git merge, vous devez vous trouver sur la branche spécifique que vous souhaitez fusionner avec votre branche parente.

Passer à la branche dans laquelle vous souhaitez fusionner tout :

$ git checkout 'branch'

Commande pour récupérer les dernières mises à jour :

$ git pull ou $ git fetch

Fusionner votre branche :

$ git merge 'branch-name'

#10 git status

git status fournit un aperçu de l'état actuel de votre référentiel.

$ git status

Il renvoie les informations suivantes concernant votre branche :

- Le nom de votre branche actuelle.
- Si votre branche actuelle est à jour ou non.
- Le nombre de validations que votre branche a en retard par rapport à sa branche distante.
- Les modifications en attente d'être validées et commises.
- Les modifications qui ne sont pas en attente de validation.
- Les fichiers non suivis.

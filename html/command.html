Les commandes Git

#1 git init

La commande git init nous permet de créer un nouveau référentiel Git. C'est la première commande pour démarrer un nouveau projet dans un référentiel GitHub. Allez dans le répertoire contenant vos fichiers de projet et exécutez la commande git init. Un sous-répertoire caché .git sera ajouté à celui-ci.

$ git init

Vous pouvez également fournir un nom de référentiel avec la commande git init.

$ git init 'nom'

#2 git clone

git clone crée une copie de travail locale du code source à partir d'un référentiel distant. Lorsque vous clonez un référentiel, le code sera automatiquement téléchargé sur votre machine. Cette commande ajoutera l'emplacement d'origine en tant qu'emplacement distant afin que vous puissiez récupérer les modifications et y pousser des changements si vous avez l'autorisation.

$ git clone 'url'

#3 git branch

La commande git branch nous permet d'ajouter une nouvelle branche à une branche existante, de visualiser toutes les branches existantes et de supprimer une branche.

Créer une nouvelle branche localement :

$ git branch 'nom branch'

Lister toutes les branches locales et distantes :

$ git branch -a

Afficher toutes les branches et voir sur quelle branche vous travaillez actuellement :

$ git branch ou $ git branch --list

Supprimer une branche :

$ git branch -d 'nom branch'

Renommer une branche :

git branch -m 'nom branch' 'nouveau nom branch'

#4 git checkout

La commande git checkout nous permet de passer à une branche existante ou de créer et de passer à une nouvelle branche. Pour ce faire, la branche vers laquelle vous souhaitez passer doit être présente sur votre système local et les modifications dans votre branche actuelle doivent être validées ou mises de côté avant de procéder au changement. Vous pouvez également utiliser cette commande pour vérifier les fichiers.

Passer à une branche existante :

git checkout 'nom branch'

Créer et passer à une nouvelle branche :

git checkout -b 'nom branch'

#5 git add

La commande git add ajoute vos modifications dans un fichier à la zone de préparation où vous pouvez comparer votre version locale et la version dans le référentiel distant. Avant de valider votre fichier nouvellement créé ou modifié, il doit être ajouté à la zone de préparation en utilisant la commande git add.

Ajouter des fichiers spécifiques :

$ git add 'nom fichier 1' 'nom fichier 2'

Ajouter tous les fichiers nouveaux, modifiés et supprimés :

$ git add -A

Ajouter les fichiers modifiés et supprimés :

$ git add -u
